## 二、库功能概述

android-stackblur 是一个高性能图像模糊处理库，实现了 Mario Klingemann 设计的 Stack Blur 算法。  
该算法是高斯模糊与 Box Blur 的折中方案：视觉效果接近高斯模糊，但比高斯模糊快约 7 倍。  
库提供三种执行路径：  
- **Java 路径**（`JavaBlurProcess`）：纯 Java 多线程实现，无 Native 依赖；  
- **Native 路径**（`NativeBlurProcess`）：通过 JNI 调用 C 语言实现，性能最优；  
- **RenderScript 路径**（`RSBlurProcess`）：使用 Android RenderScript GPU 加速，在不支持时自动回退到 Native。

### 3.2 公开接口/方法列表

#### 类：`StackBlurManager`

| 方法签名 | 参数说明 | 返回值 | 功能描述 |
|----------|----------|--------|----------|
| `StackBlurManager(Bitmap image)` | `image`：要被模糊处理的原始位图 | —（构造方法） | 保存原始图像引用，默认使用 JavaBlurProcess 作为处理器；创建 CPU 核心数大小的固定线程池 |
| `Bitmap process(int radius)` | `radius`：模糊半径（像素），有效范围 0–254 | 模糊后的 `Bitmap` | 使用纯 Java 多线程 Stack Blur 算法对保存的原始图像进行模糊，结果保存到内部 `_result` 字段并返回 |
| `Bitmap processNatively(int radius)` | `radius`：模糊半径（像素），有效范围 0–254 | 模糊后的 `Bitmap` | 使用 JNI/C 实现的 Stack Blur 对原始图像进行模糊，性能优于 Java 路径；结果覆盖 `_result` 并返回 |
| `Bitmap processRenderScript(Context context, float radius)` | `context`：Android 上下文；`radius`：模糊半径（float，内部转 int） | 模糊后的 `Bitmap` | 优先使用 RenderScript 实现对原始图像模糊；若创建 RenderScript 上下文时抛出 `RSRuntimeException`（如 ARMv6 不支持），则自动回退到 `NativeBlurProcess`；结果覆盖 `_result` 并返回 |
| `Bitmap returnBlurredImage()` | 无参数 | 上次模糊操作生成的 `Bitmap`，若未调用过任何 process 方法则为 `null` | 返回最近一次模糊结果，不触发新的模糊计算 |
| `void saveIntoFile(String path)` | `path`：目标文件系统绝对路径（如 `/sdcard/blur_output.png`） | 无 | 将最近一次模糊结果以 PNG 格式（压缩质量 90）写入指定路径；异常被 `e.printStackTrace()` 吞掉，无返回错误信息 |
| `Bitmap getImage()` | 无参数 | 构造时传入的原始 `Bitmap` | 返回原始图像的引用，不做任何处理 |

### 3.4 C 层核心函数（blur.c）

| 函数签名 | 参数说明 | 说明 |
|----------|----------|------|
| `void stackblurJob(unsigned char* src, unsigned int w, unsigned int h, unsigned int radius, int cores, int core, int step)` | `src`：RGBA_8888 格式像素数据指针（in-place 修改）；`w/h`：图像宽高；`radius`：模糊半径；`cores`：总线程数；`core`：当前线程索引；`step`：1=水平遍历，2=垂直遍历 | Stack Blur 核心算法，分解为水平/垂直两次独立 pass，每 pass 可多线程并行执行（按行或列分割）。无 Android API 依赖，纯 C 指针算术 |
| `JNIEXPORT void JNICALL Java_com_enrique_stackblur_NativeBlurProcess_functionToBlur(...)` | `bitmapOut`：Android `Bitmap` 对象（RGBA_8888 必须）；`radius/threadCount/threadIndex/round` | JNI 包装层：通过 `AndroidBitmap_lockPixels` 获取像素指针后调用 `stackblurJob`，完成后 `unlockPixels` |

---

### 3.5 查找表（blur.c，共享于 Java 和 C 实现）

| 常量名 | 类型 | 长度 | 说明 |
|--------|------|------|------|
| `stackblur_mul` | `unsigned short[]`（C） / `short[]`（Java） | 255 | 乘法因子表，索引 = radius (0–254)，用于快速整数除法模拟 |
| `stackblur_shr` | `unsigned char[]`（C） / `byte[]`（Java） | 255 | 右移位数表，索引 = radius (0–254)，与 `stackblur_mul` 配合完成归一化 |

计算公式：`blurred_channel = (sum * stackblur_mul[radius]) >> stackblur_shr[radius]`

---

### 3.6 异常与错误处理

| 情形 | 触发条件 | 处理方式 |
|------|----------|----------|
| `RSRuntimeException`（RenderScript 初始化失败） | ARMv6 等不支持 RenderScript 的设备 | `processRenderScript()` 捕获后自动回退到 `NativeBlurProcess`，并将静态标志 `hasRS` 置为 `false` 以避免后续二次尝试 |
| `IOException`（文件写入失败） | 路径不存在、权限不足等 | `saveIntoFile()` 仅 `e.printStackTrace()`，对调用方不抛出，无法感知失败 |
| `InterruptedException`（多线程等待中断） | 线程池 `invokeAll` 等待时中断 | `JavaBlurProcess.blur()` 返回 `null`；`NativeBlurProcess.blur()` 返回未完全模糊的中间 Bitmap |
| 半径超出 `stackblur_mul/shr` 数组范围 | `radius >= 255` | 数组越界，无显式校验，C 层行为未定义 |

## 六、不支持特性说明

| 特性 | Android 实现 | 不支持原因 | 建议替代方案 |
|------|-------------|------------|-------------|
| RenderScript GPU 加速 | `RSBlurProcess` + `blur.rs` | HarmonyOS 无 RenderScript API；即使是 Android 12+ 也已废弃 RS | 使用 Native C 路径（`processNatively()`），在 ARM64 上性能接近 RS |
| 多线程并行模糊 | `ExecutorService.invokeAll()` | OH ArkTS 主线程不适合直接使用 Worker 多线程操作 PixelMap | 当前版本使用单线程执行；大图场景调用方可将整个 `process()` 调度到 `taskpool` |
| `saveIntoFile(String path)` | `FileOutputStream` + `Bitmap.compress()` | 移植版聚焦核心算法，不引入文件系统依赖 | 调用方使用 `@ohos.file.fs` + `pixelMap.packing(fd)` 实现 |
